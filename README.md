Проект: WAF Service — Web Application Firewall (веб-сервис)
Срок сдачи: [12.05.2026]
Краткое описание
Веб-сервис для защиты веб-сайтов от хакерских атак. Пользователь регистрируется на сайте, получает API-ключ, подключает свой сайт через этот ключ, а затем через личный кабинет смотрит статистику атак, управляет правилами и блокировками.

Особенности
Многопользовательская система

Каждый пользователь получает свой API-ключ

WAF идентифицирует пользователя по ключу из заголовка запроса

Личный кабинет для каждого пользователя

Раздельное хранение статистики по пользователям

Асинхронный прокси-сервер на aiohttp

Что делает сервис
Регистрация и аутентификация
Пользователь регистрируется на сайте (email/пароль). После входа получает доступ к личному кабинету и свой API-ключ.

Подключение сайта к WAF
Пользователь копирует API-ключ и настраивает свой веб-сервер так, чтобы все запросы сначала шли на WAF с этим ключом в заголовке.

Фильтрация запросов
WAF принимает запрос, определяет пользователя по API-ключу, проверяет запрос на атаки по правилам этого пользователя. Если атака — блокирует (ошибка 403), если все хорошо — перенаправляет на сайт пользователя.

Личный кабинет
Пользователь заходит на сайт WAF и видит:

Количество запросов за неделю

Количество заблокированных атак

График атак по дням

Список последних атак с IP и временем

Список временно заблокированных IP с возможностью разблокировки

Список активных правил с возможностью включения/отключения

Управление правилами
Пользователь может включать или отключать правила для своего сайта. Правила: SQL-иньекция(UNION SELECT, OR/AND, Comments), XSS(скрипты, javascript протоколы), блокировка команд, блокировка сканеров.

Настройки
Пользователь может:

Настроить лимит запросов в минуту

Посмотреть инструкцию по подключению

Удалить аккаунт

Логирование
Все события (атаки, блокировки, обычные запросы) сохраняются в базе данных SQLite, привязанные к конкретному пользователю.
